# Support

Use this guide to get help with SwiftSave.

## Before opening an issue

1. Read `README.md` and `CONTRIBUTING.md`.
2. Confirm you are on a recent commit/version.
3. Collect logs and exact reproduction steps.
4. Remove secrets/tokens from logs before sharing.

## Where to ask what

- Bug reports: use GitHub Issues and select the Bug report template.
- Feature requests: use GitHub Issues and select the Feature request template.
- Security vulnerabilities: follow `SECURITY.md` and report privately.

## What to include in support requests

- Platform and OS version (Android/Linux/Windows/macOS)
- App version or commit SHA
- Exact steps to reproduce
- Expected behavior and actual behavior
- Relevant logs or screenshots (sanitized)
